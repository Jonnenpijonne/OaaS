# Evidence Model

## Purpose

This document defines what kind of operational evidence the OaaS model should produce.

Evidence is used for service review, handover, audit preparation and operational improvement.

---

## Evidence categories

| Category | Example |
| --- | --- |
| Service review | Monthly review summary |
| Incident evidence | Incident summary and follow-up actions |
| Change evidence | Approved change with validation result |
| Recovery evidence | Restore or rollback test record |
| Handover evidence | Checklist showing transfer readiness |
| Documentation evidence | Runbook or service model update |

---

## Minimum evidence quality

Useful evidence should be:

- dated
- owned
- understandable
- linked to a service or change
- free from unnecessary sensitive details
- reviewable by another person
- connected to follow-up actions when needed

---

## Evidence flow

```text
Operational activity
  ↓
Evidence record
  ↓
Service review
  ↓
Improvement action
  ↓
Updated documentation or control
```

---

## Key rule

If nobody can review what happened later, the work was not fully evidenced.
